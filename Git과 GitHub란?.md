# Git & GitHub란?




# 1. Git

  - 분산형 버전 관리 시스템이다 
  - 오픈 소스이다
  - 형상 관리 도구 중 하나이다
  - 내부적으로 보면 파일의 변화 기록을 저장하는 데이터 베이스다
  - 매 커밋마다 프로젝트 전체를 스냅샷처럼 저장한다
  - 여러 사람이 작업할때 협업 기록을 관리해준다<br>

# 2. GitHub
  - Git이라는 분산 버전 관리 시스템을 기반으로하는 웹 기반 플랫폼이다
  - 언제, 어디서나 협업 프로젝트를 쉽게 진행할수 있도록 돕는 역할을 한다



 # 3. Git & Github 주요 용어
  1.Repository: 저장소 / 말 그대로 파일이나 폴더를 저장해두는 곳이다
  2.Commit: 스냅샷 / 파일을 추가, 변경한 내용을 저장소에 영구적으로 기록하는 스냅샷이다
  3.Push: 클라우드 업로드 / 추가, 변경한 내용을 원격 저장소에 업로드하는 작업이다
  4.Pull: 최신화 업데이트 /  Github의 최신 내용을 내 컴퓨터로 다운로드 하는 작업이다
  5.Staging Area: 장바구니 / 커밋을 하기전,저장할 파일들을 골라 담아 놓는 대기 장소 이다
  6.Clone: 복제 / Github에 있는 원격 저장소를 내 컴퓨터로 그대로 복사 하여 가져오는 작업이다
