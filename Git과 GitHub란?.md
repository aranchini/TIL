# Git & GitHub란?




# 1. Git

  - 분산형 버전 관리 시스템이다.
  - 오픈 소스이다.
  - 형상 관리 도구 중 하나이다.
  - 내부적으로 보면 파일의 변화 기록을 저장하는 데이터 베이스다.
  - 매 커밋마다 프로젝트 전체를 스냅샷처럼 저장한다.
  - 여러 사람이 작업할때 협업 기록을 관리해준다.<br>

# 2. GitHub
  - Git이라는 분산 버전 관리 시스템을 기반으로하는 웹 기반 플랫폼이다.
  - 언제, 어디서나 협업 프로젝트를 쉽게 진행할수 있도록 돕는 역할을 한다.
  - 코드 변경을 리뷰하고 토론하는 기능을 제공한다.
  - 이슈 관리, 프로젝트 관리, CI/CD 자동화 같은 개발 지원 기능을 제공한다.
  - 다른 개발자의 프로젝트를 포크해서 가져와 수정하고 기여가 가능하다.
  - 팀원들이 특정 코드 라인에 댓글을 달아 토론 할 수 있다.
  - 결과를 남에게 보여주고 공유하는 '서버'역할을 한다.

# 2.1 Git Flow
  - 프로젝트의 소스코드를 관리하기 위한 브랜치 관리 전략이다.<br>
  - 여러 개발자가 협업하는 대구모 프로젝트에서 코드 엉킴을 방지하는 규칙이다.<br>
  - 역할에 따라 5가지 브랜치(Main, Develop, Feature, Release, Hotfix)를 나누어 사용한다.
>Main : 사용자에게 실제 서비스되는 완성한 코드만 관리한다.<br>
>Develop : 다음 배포를 위해 기능을 모으는 중심 브랜치이다.<br>
>Feature : 각자 맡은 새로운 기능을 개발하는 임시 브랜치이다.<br>
>Release : 배포 전 최종 테스트를 진행하고 버그를 수정하는 브랜치이다.<br>
>Hotfix : 배포된 서비스(main)에서 긴급한 오류가 발생했을 때 즉시 수정하는 브랜치이다.


 # 3. Git & Github 주요 용어
   
  **1.Repository:** 저장소 / 말 그대로 파일이나 폴더를 저장해두는 곳이다. <br>
  
 **2.Commit:** 스냅샷 / 파일을 추가, 변경한 내용을 저장소에 영구적으로 기록하는 스냅샷이다.<br>
 
 **3.Push:** 클라우드 업로드 / 추가, 변경한 내용을 원격 저장소에 업로드하는 작업이다.<br>
 
  **4.Pull**: 최신화 업데이트 /  Github의 최신 내용을 내 컴퓨터로 다운로드 하는 작업이다.<br>
  
  **5.Staging Area:** 장바구니 / 커밋을 하기전,저장할 파일들을 골라 담아 놓는 대기 장소 이다.<br>
  
  **6.Clone:** 복제 / Github에 있는 원격 저장소를 내 컴퓨터로 그대로 복사 하여 가져오는 작업이다.<br>
  
  **7.Branch :** 가지 / 기존 코드에서 독립적으로 작업할수 있도록 나누는 작업 공간이다.<br>
  
  **8. Merge:** 병합 / 서로 다른 Branch에서 작업한 내용을 하나로 합치는 작업이다.

  # 3.1 Github에서만 있는 주요 용어

  **1.Fork:** 가져오기 / 다른사람의 Github 저장소를 내 Github 계정으로 복사하는 기능이다.<br>

  **2.Pull Request:** 요청 / 내가 수정한 코드를 원래 저장소에 반영해 달라고 요청하는 기능이다.<br>

  **3.Issue:** 기록 & 관리 / 프로젝트의 버그, 문제, 기능 요청 등을 기록하고 관리하는 기능이다.<br>
  
  **4.Star:** 즐겨찾기 / 마음에 드는 프로젝트를 즐겨찾기처럼 표시하는 기능이다.<br>
  
  **5.Watch:** 알림 / 특정 프로젝트의 업데이트 알림을 받는 기능이다.<br>
  
  **6.Github Actions:** 자동 실행 / 코드 테스트, 빌드, 배포 등을 자동으로 실행하는 CI/CD 기능이다.
  > CI/CD란? 코드를 올리면 자동으로 검사하고 자동으로 배포해주는것

# 4.Git vs Github의 공통점과 차이점 

**공통점**

  - 둘다 Git이라는 동일한 버전 관리 시스템을 기반으로 동작한다.
  - 코드의 변경 이력을 기록하고, 문제가 생겼을 때 과거로 되돌리는 버전 관리가 중요하다.
  - 브랜치를 나누고 합치는 동일한 메커니즘을 사용한다.
  - 여러 사람이 함께 프로젝트를 만들 때 코드 변경을 공유하고 관리 할 수 있다.

**차이점**

| 구분 | Git | Github|
|----|---------|--------|
|종류|내 컴퓨터에 설치하는 소프트웨어|웹에서 접속하는 서비스/플랫폼|
|인터넷|오프라인에서도 작업 가능|반드스 인터넷 연결이 필요함|
|저장소| 내 로컬 PC| 원격 클라우드 서버|
|협업| 혼자서 기록을 관리함| 전세계 사람들과 코드를 공유하고 소통함|
|명령|주로 명령어를 직접 입력함|클릭으로 조작 가능한 화면을 제공함|
|보안|로컬 보안(사용자 책임)|클라우드 보안(플랫폼 관리)|
|사회성|
>GUI: 마우스 방식<br>
>CLI: 타이핑 방식
