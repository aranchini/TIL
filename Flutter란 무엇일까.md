# 1.Flutter란?
  * 구글에서 개발한 모바일 / 웹 / 데스크톱 크로스 플랫폼이다.
  * 단일 코드베이스로 안드로이드, 리눅스, 윈도우, iOS 등 여러 플랫폼에서 동작하는 앱을 만들기 위해 출시되었다.
  * 지원 언어는 Dart(다트)이다.
  * 크로스 플랫폼 앱 개발에 비해 네이티브 앱 개발에 더 가깝다.
  * Hot reload(핫 리로드)를 지원한다.
    > Hot reload란 개발자가 앱의 코드를 수정한 후 변경 사항을 즉시 반영하는데 사용된다.

# 2. Flutter의 장단점

  ### Flutter
  ___
   **장점**
 
    - 하나의 코드베이스
    - Hot reload로 빠른 UI 개발
    - 개발 속도를 높일 수 있음
    - 코드를 고치고 저장하면 굉장히 빠르게 화면에 반영됨
    - 코드 하나로 모바일 뿐만 아니라 웹, 윈도우까지 다 만들 수 있음
    - 안드로이드와 iOS 동시 개발, 앱 과 웹을 동시에 개발 가능
    - 다중 플랫폼을 지원하여 유지보수 비용을 절감할 수 있음
    - 네이티브처럼 빠르고 자연스러운 UI 제공 
    - 구글의 적극적 지원
    - Flutter는 UI 구현에 큰 장점을 가지고 있는 프레임워크
     - 퍼즐을 맞추듯 자유로운 커스텀도 가능 
    
   **단점**
    
    - OS 기능 접근엔 제한 있음
    - 네이티브 앱에 비해 초기 앱 용량이 비교적 큰 편
    - 기기의 깊숙한 기능을 쓸때는 네이티브 코드를 만질 줄 알아야함
    - iOS 나 안드로이드 버전이 새로 나왔을때 외부 라이브러리 제작자가 업데이트 안해주면 내 앱에서 에러 발생할 수도 있음
    - 생소한 Dart 언어로 추가학습 필요
    - 네이티브 기능 활용시 별도 플러그 인 필요
    - 생태계가 아직 성장중
    - 구글이 갑자기 전략을 바꾼 사례가 많아 장기적인 지속 지원성에 대해 불안감을 가진 개발자도 있음
    - Flutter는 통합 위젯이라서 각 플랫폼 고유의 UX를 살리기 어려움
    
# 3. Flutter와 네이티브의 차이점

| 구분 | Flutter | 네이티브 |
|----|----------|---------|
|개발방식| 하나의 코드로 안드로이드와 iOS를 동시에 개발| 안드로이드와 iOS를 따로 개발|
|개발 속도| Hot Reload로 빠른 개발| 플랫폼 별로 따로 개발하여 느림|
|언어| Dart| Swift(iOS), kotlin / Java(안드로이드)|
|앱 크기| 상대적으로 큼|가장 작음|
|사용 기준| 빠른 개발과 멀티 플랫폼 서비스에 유리 | 최고 성능과 플랫폼 최적화 필요 서비스에 유리 |
|유지보수| 하나의 코드베이스로 여러 플랫폼 유지|플랫폼 별로 따로따로 관리해야함|
|대표 활용 사례|구글 광고앱, BMW등|카카오톡,네이버,배달의 민족|

# 4. Flutter와 리액트 네이티브의 공통점 과 차이점

**공통점**

  - 두개 다 크로스 플랫폼
  - 모두 우수한 성능을 보여줌
  - 두개 다 오픈소스임
  - Hot reload 지원
  - UI 화면 구성 가능
  - 네이티브 기능 사용 가능
  - 생산성 및 비용 절감
  - 둘 다 라이브러리가 풍부함
  - 두개 다 개발 속도 빠름

  **차이점**

|구분 | Flutter | 리액트 네이티브| 
|-----|---------|----------------|
|언어| Dart|JavaScript|
|성능| 계산이 복잡한 작업 수행시 CPU 사용률이 증가함| 자체 렌더링 기능이 있어 CPU 사용량이 상대적으로 낮음|
|장점|- 시스템,코드 작성이 직관적임 <br> - 기본 제공되는 툴의 퀄리티가 높음 <br> - 네이티브 앱 못지않은 속도와 성능을 자랑함|- 커뮤니티 규모가 크고 개발자 인력 풀이 넓음<br>- 라이브러리가 다양하고 많음|
|UI 구성 방법| 자체 위젯 렌더링 엔진 사용 | 네이티브 UI 브리지로 연결해 사용|
|플랫폼 지원 범위| 안드로이드, iOS뿐만 아니라 웹,윈도우 리눅스까지 가능|기본적으로 안드로이드,iOS에 집중하고 있음|
|디버깅 과정| 쉬움|다소 어려울수 있음|
|대표 활용 사례| GOMS,Toss,알리 바바|디스코드, 인스타그램 , 테슬라|


### 용어정리

- **단일 코드베이스:**
 하나의 코드로 여러 플랫폼에서 실행되는 프로그램을 만드는 것<br>
- **오픈소스:**
  소스 코드를 누구나 볼수있고 수정하고 사용할 수 있도록 공개한것<br>
- **라이브러리:**
  이미 만들어져 있는 코드 모음<br>
- **자체 렌더링 기능:**
  화면을 그리는 일을 프레임워크가 직접 하는 것<br>
- **프레임 워크:**
  프로그램을 만들 때 기본 구조와 기능을 미리 만들어 둔 개발 도구<br>
- **디버깅:**
  프로그램의 오류를 찾아 수정하는 과정
